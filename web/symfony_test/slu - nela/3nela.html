11. Doctrine > Database Abstraction Layer
Multiple choices

Repository

What is wrong with this repository method ?

PHP
public function filterBooks(array $filters)
{
    $qb = $this->createQueryBuild('b');
    $qb->where('b.year = :year');
    $qb->setParameters(['year' => $filters['year']]);

    $qb->andWhere('b.gender = :gender');
    $qb->setParameters(['gender' => $filters['gender']]);

    return $qb->getResults();
}

Wrong

  • The second call setParameters(...) will erase data set in the first call. So year will be deleted from the parameters list.
    You should use setParameter($name, $value).

  • $qb is never updated after its initialization.
    You should do methods chaining.

  • The method getResults does not exists.
    You should use getResult().

The choices were

  • The second call setParameters(...) will erase data set in the first call. So year will be deleted from the parameters list.
    You should use setParameter($name, $value).

  • The method getResults does not exists.
    You should use getResult().

  • $qb is never updated after its initialization.
    You should do methods chaining.

  • The query builder does not have a getResults() method.
    You should use getQuery().

Help

https://symfony.com/doc/current/doctrine.html#querying-for-objects-using-doctrine-s-query-builder

The "with" tag

Consider the following Twig snippet:

Twig
{% with %}
    {% set maxItems = 7 %}
    {# ... #}
{% endwith %}

{# ... #}

{% for i in 1..maxItems %}
    {# ... #}
{% endfor %}

Will the Twig template work as expected?

Correct

  • No. The template will display an error because the maxItems variable is not defined outside the with tag.

The choices were

  • No. The template will display an error because the with tag is not defined.

  • No. The template will display an error because the maxItems variable is not defined outside the with tag.

  • No. The template won't iterate from 1 to 7. It will execute the for loop just one time (where i is 1).

  • Yes

Help

Twig 1.28 (November 2016) introduced a new with tag that allows to create different scopes in the same template. Variables defined inside a with tag are not available outside of it. See http://twig.sensiolabs.org/doc/tags/with.html

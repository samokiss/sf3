Kernel Interface

Is there any problem in the following code?

PHP
// app/AppKernel.php
use Symfony\Component\HttpKernel\Kernel;

class AppKernel extends Kernel
{
    public function registerBundles()
    {
            $bundles = [];

            if ('prod' === $this->getEnvironment()) { 
                $bundles = [
                    new Symfony\Bundle\FrameworkBundle\FrameworkBundle(),
                 ];
            }

            if (in_array($this->getEnvironment(), array('dev', 'test'))) { 
                $bundles[] = new Symfony\Bundle\WebProfilerBundle\WebProfilerBundle();
            }

            return $bundles; 
    }

    // ...
}

Correct

  • Yes, the dev and test environments won’t work properly.

The choices were

  • Yes, the registerBundles() method must return an object implementing the BundleListInterface, not an array.

  • No.

  • Yes, the dev and test environments won’t work properly.

  • Yes, the WebProfilerBundle bundle should be enabled in the prod environment too.

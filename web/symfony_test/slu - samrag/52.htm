Single choice

setDefault with callable

With the following code:

PHP
<?php
// ...
class Mailer
{
    // ...
    public function configureOptions(OptionsResolver $resolver)
    {
        // ...
        $resolver->setDefault('encryption', null);

        $resolver->setDefault('port', function (Options $options) {
            if ('ssl' === $options['encryption']) {
                return 465;
            }

            return 25;
        });
    }
}

what would be the resolved $options passed as:

PHP
$options = array('encryption' => 'SSL');

Correct

  • PHP
    $options = array(
    'encryption' => 'SSL',
    'port' => 25,
);

The choices were

  • PHP
    $options = array(
    'encryption' => 'SSL',
    'port' => 25,
);
  • PHP
    $options = array(
    'encryption' => 'SSL',
    'port' => 465,
);
  • PHP
    $options = array(
    'encryption' => 'SSL',
    'port' => function () {
            if ('ssl' === $options['encryption']) {
                return 465;
            }

            return 25;
        },
);
  • PHP
    $options = array(
    'encryption' => 'SSL',
    'port' => null,
);

Help

  • http://symfony.com/doc/current/components/options_resolver.html#default-values-that-depend-on-another-option

The "with" tag

Consider the following Twig snippet:

Twig
{% with %}
    {% set maxItems = 7 %}
    {# ... #}
{% endwith %}

{# ... #}

{% for i in 1..maxItems %}
    {# ... #}
{% endfor %}

Will the Twig template work as expected?

Wrong

  • Yes

The choices were

  • No. The template won't iterate from 1 to 7. It will execute the for loop just one time (where i is 1).

  • No. The template will display an error because the with tag is not defined.

  • No. The template will display an error because the maxItems variable is not defined outside the with tag.

  • Yes

Help

Twig 1.28 (November 2016) introduced a new with tag that allows to create different scopes in the same template. Variables defined inside a with tag are not available outside of it. See http://twig.sensiolabs.org/doc/tags/with.html
